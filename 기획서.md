### 7.1 목표·전환 방식

| 항목 | 내용 |
|------|------|
| **목표** | 스마트폰·태블릿에서 가독성·터치 조작이 가능한 **단일 코드베이스** 반응형 |
| **PC 유지** | `responsive: false` 시 **현재 1280px 고정 레이아웃 그대로** (클라이언트 PC 검수용) |
| **모바일 활성화** | `js/site-config.js` → `responsive: true` + `js/viewport.js`가 `desktop-only` 클래스 제거 |
| **원칙** | PC 확정 CSS는 **건드리지 않음**. 모바일은 기존 `html:not(.desktop-only)` 미디어쿼리 **보완·추가** |

### 7.2 브레이크포인트 (기존 CSS 기준)

| 구간 | 너비 | 용도 |
|------|------|------|
| **Desktop** | ≥ 1024px | PC GNB 가로 메뉴 + 호버 서브바 |
| **Tablet** | 768px ~ 1023px | 햄버거 메뉴, 2열 그리드 위주 |
| **Mobile** | ≤ 767px | 1열 스택, 좌측 서브메뉴 → 가로 탭/스크롤 |
| **Small** | ≤ 520px | 실적 1열, 사업 카드·폼 여백 축소 |

> 이미 `styles.css`에 `html:not(.desktop-only)` 조건 미디어쿼리 **약 36곳** 존재. `responsive: true` 전환 시 **즉시 일부 적용**되나, 메인·푸터·게시판 등 **추가 작업 필요**.
